# HK 허브 디자인 정의 — 「HK 여권 · 중앙 발권국」

> 5개 앱 디자인 시스템(HK-UI 쇼케이스)을 참고해 **허브만의 테마 `theme-hub`**를 정의한다.
> 공유 골격은 그대로 쓰고, 색 팔레트와 시그니처 물성만 허브 고유로 추가한다.

## 1. 참고한 디자인 시스템 (HK-UI)

하나의 네오브루탈리즘 시스템("갱지 위 인쇄 + 형광 오프셋 그림자")이 **앱별 색 테마로 갈아끼워지는** 구조.

**공유 골격 (허브도 그대로 승계 — 절대 바꾸지 않음)**
- 디스플레이: `Cinzel` + `Noto Serif KR` (serif) · 본문: `Noto Serif KR`/`Pretendard` · 숫자: `Space Mono`
- 테두리 `2px solid` 잉크 · 라디우스 `3px`(거의 각짐) · 하드 오프셋 그림자 `3px 5px 0`
- 텍스처: grain(turbulence) + halftone(radial-dot 7px) · 이징 `thunk`(튕김)·`settle`
- 스페이싱 4→48 스케일 · 숫자 크기 `--num-sm..xl`(14/20/36/48)

**앱별 시그니처 물성(物性)과 팔레트**

| 앱 | 물성 | 표면 | accent / accent-2 |
|---|---|---|---|
| 증권가 stock | 시세 차트·전광판 | 밝은 종이 #e9e5da | 핫핑크 #ff2d6b / 블루 #2b52ff |
| 베팅판 betting | 발권 티켓 | 밤 #14100c | 앰버 #f0a92b / 블루 #4aa8ff |
| 교환소 dp | 영수증 | 민트 종이 #e8ece6 | 틸그린 #12b886 / 블루 |
| 심판소 judge | 채점 도장 | 크림 #efe7d6 | 적갈 #7c2b23 / 슬레이트 #2f5d86 |
| 의뢰소 request | 현상수배지 | 크래프트 #3a2c1e | 핏빛 #9c1f1a / 브론즈 #b8862b |

## 2. 허브의 자리 — 왜 「여권 · 발권국」인가

5개 앱은 각자 물성을 가진 **장소**다(거래장·베팅장·교환소·심판소·의뢰소). 허브는 그 장소들과 나란한 6번째 앱이 아니라 **그 위의 기관**이다:
- 5개가 공유하는 포인트를 발권·관리하는 **중앙 발권국**
- 모든 홀에서 통하는 **단일 신분증(여권)** — 로그인 = 여권 제시
- 각 홀의 티켓·도장·영수증을 *모으는* 메타 물성 → 5개와 겹치지 않는다

그래서 시그니처 물성 = **여권(passport) + 관인(official seal)**. 화면은 여권 펼침면 은유:
- 표지·신원면 = 이름 + 순자산(발권국이 보증하는 총 잔고)
- 통화면 = 포인트 · DP · 주식 평가액 (보유 통화 목록)
- 입국 도장면 = 의뢰·봉사 현황 (다녀간 홀의 기록 = 고무도장)
- 비자/디렉토리면 = 5개 홀 바로가기 — **각 홀을 그 홀의 고유 색으로** 표시(허브가 5색의 목차가 됨)

## 3. `theme-hub` 팔레트

5개가 주색으로 안 쓴 **딥 네이비(관인·관공서)**를 accent로, 경제를 상징하는 **골드**를 공유 가치색 accent-2로. 서늘한 종이 위 네이비 잉크 → 중립·상위·공적.

```css
body.theme-hub {
  --surface-base:   #dfe3ee;  /* 서늘한 청회색 종이 — 로비 바닥/여권 표지 안쪽 */
  --surface-raised: #f0f2f8;  /* 카드·면 — 차가운 흰 종이 */
  --surface-sunken: #d3d8e8;
  --surface-invert: #171b2e;  /* 네이비-블랙 — 표제부·관인 바탕 */
  --ink-primary:    #1a1f36;  /* 네이비 잉크 본문 */
  --ink-secondary:  #5a6180;
  --ink-on-invert:  #e7ebf6;
  --accent:         #2a3a8f;  /* 딥 인디고 — 관인/발권 인장 (5개 미사용 주색) */
  --accent-2:       #b8862b;  /* 골드 — 공유 통화·가치 (economy를 governs) */
  --highlight:      #ffd23f;  /* 공유 하이라이트 */
  --gain:           #9c1f1a;  /* 공유: 이익 적색 */
  --loss:           #3a5b7a;  /* 공유: 손실 슬레이트 */
}
```

골격 토큰(폰트·테두리·라디우스·그림자·텍스처·이징)은 `:root` 공유본을 그대로 사용. 허브는 색 6~10줄만 오버라이드한다.

## 4. 허브 고유 컴포넌트 (5개 시스템 위에 얹는 것)

공유 버튼·카드·핀·배지·스탯은 그대로 쓰되, 허브만의 시그니처 3종을 추가한다.

1. **관인 헤더(Bureau masthead)** — `--surface-invert` 바 + Cinzel `HK · 여권` / `CENTRAL BUREAU OF POINTS`, 우측에 원형 **관인 인장**(2겹 원 + 별, `--accent` 잉크, halftone 텍스처). 로그인 화면의 표지.
2. **통화면 스탯(Currency ledger)** — 순자산을 `--num-xl` Space Mono 대형으로, 그 아래 포인트/DP/주식평가액을 "보유 통화" 3행. 배경에 halftone 살짝. (기존 대시보드 4스탯을 이 은유로 재배치)
3. **입국 도장(Entry stamp)** — 의뢰·봉사 현황을 살짝 기운(−4°) 고무도장 배지로. 상태색은 공유 토큰(모집중=accent, 완료=positive, 분쟁=gain). 심판소 도장 모티브를 "다녀간 기록"으로 전용.
4. **홀 디렉토리(Visa directory)** — 5개 홀 카드를 각 홀의 **고유 accent 색 탭/도장**으로. 허브만 네이비 중립이라, 5색이 허브 안에서 목차처럼 정렬된다. → 통합감의 핵심 장치.

## 6. 원칙

- 허브는 **6번째 색 테마**를 추가할 뿐, 시스템을 포크하지 않는다(골격 공유).
- 허브 accent(네이비)는 5개 어디서도 주색이 아니어야 한다 → 상위·중립성 유지.
- 각 홀은 허브 안에서 **자기 색**으로 나타난다 → 허브 = 5색의 목차.
- 카피는 발권국 톤(공적·간결): "여권 제시", "보유 통화", "입국 기록".
