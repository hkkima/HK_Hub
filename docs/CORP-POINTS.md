# 팀 포인트 (Team Points) — 설계 + 구현 문서

> **팀 단위 통화.** 개인 포인트와 같은 단위지만 팀 금고에 동결(DP 전환 불가).
> ✅ **2026-07-20 구현·배포·집행 완료.** 확정 결과는 §12, 실행 절차는 [RELISTING-RESET.md](./RELISTING-RESET.md).

## 0. ★팀 = 주식★ (가장 중요한 구조)

**별도 `companies` 컬렉션은 없다.** 팀의 실체는 상장 종목 문서다:

| 항목 | 위치 |
|---|---|
| 사명·팀원 | `stocks/{id}.name` · `.members` (기존 필드, 자사주 매수제한·스톡옵션에 사용) |
| **대표(CEO)** | `stocks/{id}.ceoUserId` |
| **팀 금고** | `stocks/{id}.corpBalance` |
| 팀 원장(공개) | `teamLedger/{id}` (`stockId` 키) |
| 서비스 주문 | `corpOrders/{id}` |

- **상장 = 팀 생성** (`upsertStock`), **상장폐지 = 팀 해산** (`delistStock` — 잔여 금고를 housePool로 회수해야 총량보존)
- 역할 분담: **상장·대표·팀원 지정 = HK_Stock 관리자** / **금고 충전·주급·상여·배당 = 허브**

## 1. 확정된 파라미터

- **주체 = 신규 4사** (8팀 → 4사 재편, 리브랜딩, 팀원 완전 재배치)
- **지출 권한 = CEO 1인** (대표가 팀 금고 집행, 함수가 `ceoUserId` + PIN 검증)
- **까미 = 3-in-1** (AI 노동 + Q&A/피드백 자문 + 브랜드/IP 엔도스먼트)
- **기업 포인트는 DP 전환 불가** — 역량 구매(소각) + 월급(개인 환류)의 두 출구만. (전환 허용 시 R0 방파제 우회로가 되어 설계 붕괴)
- **월급제** — 기업 금고 → 개인 포인트 정기 지급, CEO가 책정. 동기부여이자 **현금 유출 속도 밸브**.
- **DP 곡선 R0 리베이스** — 리셋 시 명목 인플레에 맞춰 R0 상향(현금 노출 완충). AMM 초기화와 동시.
- **개인→기업 유입 = IPO 청약 자본금 + 거래세(매도수수료 분할) + 자발 출자** (§8 경제모델).

## 2. 설계 논지 (동기부여)

**개인 포인트 = 보상(소비) / 기업 포인트 = 재투자(역량).**
- 기업 포인트는 "팀이 함께 번 금고" → 협업·집단 책임·자원 배분 학습
- **역량 재투자에 쓰게** 설계 → 선순환: 성과 → 기업 포인트 → 역량 구매 → 더 나은 성과
- 수강생 = 게임 기획 트레이니 → **메커니즘 자체가 교보재**(기회비용·투자 vs 소비·엔도스먼트 경제)

## 3. 획득 (재상장 이벤트)

이벤트 순서:
1. 목(장마감) → 금·토·일 장 close
2. 8개 팀 프로젝트 최종평가 → 순위
3. 순위별 **배당** 지급(개인)
4. **전 보유주식 공정가 청산** → 개인 환수 (`보유주 × 현재가`, AMM 하락 미적용) — 사실상 **하드 리셋**, 실행계획은 [RELISTING-RESET.md](./RELISTING-RESET.md)
5. 신규 4개 합병기업 **IPO + IR** → IR 순위
6. **(기존 8 순위 + 신규 4 순위) 합산 → 기업 포인트 지급**

> 이후 지속 획득원(선택): 분기 실적(시세·배당)·IR·프로젝트 마일스톤 등으로 정기 지급.

## 4. 쓰임새 카탈로그 (동기 레버별)

### ① 역량 재투자 (유능감) — "회사를 더 잘 굴리게"
- **까미 노동**: 리서치·에셋·코드리뷰 태스크 대행 (시간 확보) · 소각
- **강사 개입권**: 1:1 컨설팅·페어링·기획리뷰 예약 슬롯 (희소→전략적) · 소각
- **리소스 지원**: 에셋팩·툴 라이선스·크레딧·데이터 열람권 · 소각
- **테스터 의뢰**(의뢰소 연동): 수강생을 유저테스터로 고용 (외부 피드백 + 학급 순환)

### ② 까미 자문 (성장) — "막힘을 뚫게"
- **까미 우선 Q&A / 심층 피드백**: kami-qa-bot 한도 상향·우선응답 · 소각

### ③ 까미 브랜드 · 시장 (지위) — "회사를 알리게"
- **까미 엔도스먼트(홍보 대행)**: 까미가 자사 홍보 뉴스·공지 게시 (인지도·시세 영향)
- **까미 IP 사용권**: 굿즈·로고 콜라보에 까미 캐릭터 사용
- **까미 앰버서더 독점**: 기간 독점 파트너 (희소·고가 → 회사 간 입찰 경쟁)
- **IR/호재 뉴스 슬롯**: 자체 홍보 뉴스(남발 방지 위해 고가+쿨다운)
- **브랜딩**: 로고·컬러·리더보드 명패·회사 배지
- **추가 상장/신사업**: 자회사 상장·신규 티커

### ④ 팀 배분 (관계·자율) — "번 걸 나누게"
- **팀 배당**: 기업 포인트 → 팀원 개인 포인트 환급(성과급). **CEO가 배분 결정** → 리더십·공정성 드라마
- **마일스톤 보너스 풀**

### ⑤ 우선권·전략 (우위)
- 의뢰소 우선노출 · 채용 우선 · 강사 슬롯 우선예약 · 숨은 지표(종목 trait·시장데이터) 열람

### ⑥ 성장·해금 (숙달)
- **기업 티어**(스타트업→중견→대기업): 누적 기업 포인트가 회사 레벨, 상위 기능 해금 (일회성 아닌 성장곡선) · 비소비 누적

**동기 튜닝 노트**
- 역량재(①)는 비싸게·신중히 / 지위재(③)는 값싸게·자주 → 경제 리듬
- 고레버리지(강사 개입·까미 독점)는 희소·고가 → 매 순간 "쓸까 아낄까"
- 투자 ROI > 소비 ROI 로 책정 → "역량에 쓰는 게 이득" 학습
- **소각(sink) 위주** → 인플레 통제, housePool 회계(예산 편성)와 직결

## 5. 거버넌스 = CEO 1인 + 투명성 견제

- 지출은 CEO만. 함수가 CEO 신원 검증.
- 하드 거버넌스 대신 **공개 원장(teamLedger read:if true)** 으로 견제 — 모든 지출을 전 학급이 봄 → 사회적 책임이 규칙을 대신(기존 "학급 투명성" 패턴 재사용).
- 팀 배당도 CEO 결정 + 공개 로그.

## 6. 경제·무결성

- **재원**: 이벤트 시 housePool 민팅(faucet). 대부분 사용처는 소각(sink) → 순인플레 통제.
- **기업→개인 이전**(팀 배당)은 개인 balance 증가 → **반드시 Cloud Functions 경유**(불변식: 개인 balance 증가는 함수만).
- housePool은 주식·DP·저지·봉사와 공유 → 기업 포인트 민팅/소각도 **전 ledger type 회계에 포함**(예산 편성 뷰에 반영).

## 7. 관리 대시보드 · 백엔드 접점

- **엔티티**: ★팀 = 주식★ — `stocks/{id}` 에 `ceoUserId`·`corpBalance` 를 더해 팀을 표현(별도 companies 없음). 원장은 `teamLedger`. 상장=팀 생성, 상폐=팀 해산(잔여 금고는 housePool 회수).
- **재상장 이벤트 콘솔**(1회성 워크플로우): 프로젝트 순위 입력 → 배당 → **공정가 청산** → IPO 등록 → IR 순위 → 기업 포인트 지급.
  - ⚠️ **공정가 청산**이 최고 난도: 전 종목 `보유주×현재가`를 슬리피지 없이 개인 환수 + housePool 총량보존. 별도 1회성 정산 함수로 설계·검증 필요(자동 매도 아님).
- **기업 포인트 집행**: 자동집행(까미 노동·팀배당=함수) vs 강사 수동승인(강사개입·리소스·까미 엔도스먼트=승인 큐). 의뢰소 `helpRequests` 승인 패턴 재사용.
- **공개 원장 뷰**: teamLedger를 학급이 볼 수 있게(투명성 견제).

## 8. 경제 순환 모델 (전체 설계)

기업 레이어는 개인 유동성을 흡수·동결하는 **싱크**다. 명목 포인트 인플레는 커도, 현금으로 새는 속도는 이 순환이 잡는다.

```
  [개인 포인트] ──① 유입(IPO청약·거래세·출자)──▶ [기업 금고]  ← DP 전환 불가(동결)
       ▲                                              │  │
       │④ DP→현금 (R0 리베이스가 속도 완충)             │  │② 소각: 까미·강사·리소스·브랜드
       │                                              │  ▼   (경제에서 이탈)
       └──────────── ③ 월급(CEO 밸브) ────────────────┘
```

**4개 흐름**
- **① 유입 (개인→기업)**: 개인 유동성을 금고로 흡수 → 현금 전환 대상에서 제외(동결)
- **② 소각 (기업→서비스)**: 까미·강사·리소스·브랜드 구매 = 경제에서 영구 이탈(디플레)
- **③ 환류 (기업→개인)**: 월급. CEO가 속도 조절 = **현금 유출 밸브**
- **④ 유출 (개인→DP→현금)**: 유일한 현금화 경로. R0가 단가 방파제

**현금 노출 공식**: `현금상한 ≈ (개인 유동 ÷ R0) × 500원`.
개인 유동 = 리셋지급 − 흡수(①) + 월급(③). → **운영자가 ①③과 R0로 현금을 직접 통제**.

### 8.1 개인 → 기업 유입 (빈칸 설계)

3층으로 흡수. **흡수율이 곧 현금 노출을 정한다.**

1. **IPO 청약 자본금** (일회성·대량 흡수) — 리셋 후 4사 IPO 시 청약금을 분할:
   - 대부분 → 본딩커브 `reserve`(거래 가능 지분, 본전보장 유지)
   - 공시된 일부(예 20~30%) → **기업 금고**(paid-in capital, 매도 시 반환 안 됨 = 실제 지분 성격)
   - 이 분할 비율 = **주 흡수 다이얼**. 리셋 유동성(+171% 상한)을 여기서 대량 흡수.
   - ⚠️ 구현: reserve<mark 갭이 의도적으로 생김 → 트레저리 바이백을 reserve 한도로 하거나 금고에서 보전할지 결정(§10).
2. **거래세 = 매도수수료 분할** (지속·저위험 다이얼) — 기존 2% 매도수수료를 `1% housePool + 1% 자사 금고`로 분할. 신규 메커니즘 0, 곡선 무영향. 거래 활발한 회사가 금고↑(인기 종목 보상).
3. **자발 출자** (보조) — 개인이 팀 금고에 직접 후원(인정·월급 우선권 등 반대급부). 자율·소속감.

> "지금 포인트도 어느정도 팀으로 흐른다" = ① 구조가 만드는 자동 흐름. **주식 매수가 곧 자기 팀 자본 조달**이 되어 투기 이상의 의미를 얻는다(IPO=자본조달, 교육적 정합).

### 8.2 월급제 (환류 밸브 + 동기)

- **정기성 = 안정·소속감**: 한 방 배당보다 매주 수입이 "이 회사 직원" 정체성. 현금화 압력↓(자연 방파제)
- **CEO가 책정**: 기본급/성과급 배분이 리더십의 실체. 공개 원장으로 전 학급이 감시
- **체불 압박 = 경영 트레이드오프**: 금고 비면 월급 불가 → **역량 투자 vs 인건비** 진짜 경영 딜레마(가장 값진 학습 지점)
- **경제 역할**: 월급률 = 현금 유출 속도. 금고 동결분은 풀린 만큼만 현금화 가능

### 8.3 R0 리베이스 (현금 방파제)

- 명목 포인트가 ~2–3배로 부푸는데 `R0=10,000` 고정이면 DP가 상대적 헐값 → 현금 유출 가속.
- 리셋 시 `R0 ← R0 × 명목인플레계수`로 리베이스(권장 시작값 **25,000~30,000**, 다이얼). AMM 초기화와 동시 수행.
- 효과(실측): 유동 절반 흡수 + R0 3배 → 현금 노출 **약 1/6**.

## 9. 운영자 다이얼 (현금 예산 통제판)

| 다이얼 | 무엇을 정함 | 대시보드 노출 |
|---|---|---|
| **흡수율**(IPO 분할·거래세율) | 개인→기업 흡수량 = 동결 규모 | 예산 편성 뷰 |
| **월급률** | 현금 유출 속도 | 기업 콘솔 |
| **R0** | DP 단가 = 현금 전환 완충 | DP 설정 |
| **소각 가격표** | 까미·강사·리소스·브랜드 가격 = 소각 속도 | 기업 콘솔 |

네 다이얼이 §8 공식에 들어가 **현금 노출·인플레를 실시간 조절**. 예산 편성(포인트→DP→현금) 뷰가 이걸 시뮬레이션.

## 10. 남은 결정 (착수 전)

- 획득 공식: (기존8 + 신규4 순위) → 기업 포인트 환산식(선형/가중)
- **IPO 청약 분할비**(금고 %) 초기값 + reserve<mark 바이백 처리
- **거래세 분할비**(housePool/금고)
- **월급률·주기** 기본값, CEO 조정 범위
- **R0 리베이스 목표값** 확정(25k~30k 권장)
- 티어(⑥) 도입 여부, 사용처 가격표(까미 독점·강사 희소가), 엔도스먼트 시세영향 엔진

## 11. 팀 경제 시스템 명세 (구현 대상)

재상장 후 활성화. 전제: ★팀 = 주식★ — `stocks/{id}`(name·team·members·**ceoUserId**·**corpBalance**) + `teamLedger`(공개). 상장·대표·팀원 지정은 HK_Stock 관리자(upsertStock), 금고 충전·지급은 허브. 모든 지출은 **CEO만**(함수가 ceo 검증) + **공개 원장**. 잔고 변경은 `FieldValue.increment`.

### 11.1 주급 + 소득세 (핵심)
- `paySalary({stockId, ceoUserId, pinHash, payments:[{userId, gross}]})` — 각 팀원에게 gross 지급 시 **10% 원천징수**: 팀원 `+gross×0.9`, housePool `+gross×0.1`, 팀 금고 `−gross`. 원장에 gross/tax 기록.
- **세금이 housePool로 감** → 리셋으로 생긴 **housePool −296K 적자를 점진 회수**하는 자연 장치(설계 보너스).
- 주 마무리 정기 지급. 금고 부족 시 실패(체불 압박).

### 11.2 상여 / 자체 배당
- `payBonus({stockId, ceoUserId, pinHash, userId, amount})` — 수시. **소득세 15% 원천징수**(정기급여인 주급 10%보다 높게 → 정기 지급 우대). 팀원 `+amount×0.85`, housePool `+amount×0.15`.
- `payTeamDividend({stockId, ceoUserId, pinHash, perShare})` — 자사주 보유자에게 `perShare×shares` 지급(팀 금고 재원). **자사주 보유 동기** ↑. 세금 없음(우선).

### 11.3 팀 포인트 교환소 (②소각)
- `redeemCorpService({stockId, ceoUserId, pinHash, service, params})` — 팀 금고 차감 + 이행 레코드 생성.
  - **까미 일일 고용**(work 대행) · **까미 테스터 계약**(유저테스트) · **강사 지원 계약**(컨설팅/리뷰) 등.
  - 자동집행(까미) vs 강사 수동승인(강사지원) 혼합 — 의뢰소 `helpRequests` 승인 큐 패턴 재사용.
  - 가격표 = 소각 다이얼(§9).

### 11.4 스톡옵션 개선 (설계만)
현행: `holdings.locked`(매도 불가 잠금분)만 존재. 개선 방향(미구현):
- **CEO 부여**: 팀원에게 옵션 grant(수량·행사가·베스팅 조건).
- **베스팅**: 출결/마일스톤/기간 충족 시 단계 해금(locked→free).
- **행사(exercise)**: 행사가로 매수 권리 or 현재가−행사가 차액 정산.
- **동기**: 리텐션·성과 연동. 무상 grant vs 행사가 방식, 베스팅 트리거를 재상장 후 확정.

### 11.5 세금의 확장 여지 (선택)
- 우선은 주급 소득세 10%만. 이후 거래세(§8.1-2)·배당세·교환소 부가세 등으로 확장 가능 — 전부 housePool 회수 경로 → 적자 치유 + 예산 편성 뷰 입력.
